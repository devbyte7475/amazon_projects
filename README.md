# Amazon Payment 数据处理工具 - 使用说明

## 📋 功能特性

### ✨ 核心功能
- **单文件设计**：所有功能集成在一个 Python 脚本中，无需额外依赖文件
- **拖放操作支持**：直接将数据文件夹拖放到终端窗口即可开始处理
- **智能数据合并**：自动识别并合并 Excel (.xlsx/.xls) 和 CSV 文件
- **跨系统兼容**：完美支持 macOS 和 Windows 系统
- **智能编码识别**：自动处理不同编码的 CSV 文件（GBK, UTF-8, Latin-1等）
- **数据类型转换**：智能识别并转换数值类型列
- **业务指标计算**：自动计算销量、退货量、销售额等业务指标

### 🛡️ 增强功能
- **进度指示器**：实时显示处理进度（需安装 tqdm）
- **日志记录**：完整的处理日志，便于问题追踪
- **错误处理**：健壮的异常处理机制，不会因单个文件错误而中断
- **输入验证**：自动验证文件夹路径和文件有效性
- **处理摘要**：处理完成后显示详细的统计信息

## 🚀 快速开始

### 1. 环境要求

```bash
Python 3.7+
pandas
numpy
openpyxl  # 用于读取 .xlsx 文件
xlrd      # 用于读取 .xls 文件
tqdm      # 可选，用于显示进度条
```

### 2. 安装依赖

```bash
pip install pandas numpy openpyxl xlrd tqdm
```

### 3. 运行脚本

#### 方法 1：拖放文件夹（推荐）
1. 打开终端
2. 运行脚本：
   ```bash
   python payment.py
   ```
3. 将数据文件夹直接拖放到终端窗口
4. 按回车键开始处理

#### 方法 2：手动输入路径
1. 运行脚本
2. 手动输入文件夹完整路径
3. 按回车键开始处理

### 4. 输出结果

处理完成后，会在数据文件夹中创建 `processed_results` 子文件夹，包含：
- `payment_原表_逻辑化.xlsx`：处理后的原始数据表

## 📊 数据文件要求

### 支持的文件格式
- Excel: `.xlsx`, `.xls`
- CSV: `.csv`

### 文件内容要求
- 第一行为表头（或跳过指定行数）
- 必须包含以下列（不区分大小写）：
  - `date/time` 或 `日期时间`
  - `type` 或 `类型`
  - `sku`
  - `quantity` 或 `数量`
  - `product sales` 或 `产品销售额`
  - `total` 或 `总计`
  - 其他业务相关列

## 🔧 高级配置

### 修改跳过行数

如果您的数据文件表头不在第一行，可以修改代码中的跳过行数：

```python
df = self.datamerge(folder_path, skip_csv=7, skip_excel=7)
```

- `skip_csv`: CSV 文件跳过的行数
- `skip_excel`: Excel 文件跳过的行数

### 自定义输出路径

修改 `export_results` 方法中的输出文件夹名称：

```python
output_folder = folder_path / 'processed_results'  # 修改此处
```

## 📊 处理流程

1. **数据合并** (`datamerge`)
   - 扫描文件夹中的所有数据文件
   - 自动识别文件编码
   - 合并所有数据到一个 DataFrame

2. **数据类型转换** (`datatransform`)
   - 智能识别数值列
   - 转换数据类型（Int64/Float64）
   - 处理百分数、货币符号等

3. **日期处理** (`process_dates`)
   - 解析日期字符串
   - 提取年、月、日

4. **数据维度划分** (`add_data_dimensions`)
   - 根据 SKU 是否为空划分数据维度
   - 店铺维度 vs 商品维度

5. **业务指标计算** (`create_metric_columns`)
   - 计算销量、退货量、销售额等
   - 计算预估货值和预估毛利

6. **结果导出** (`export_results`)
   - 导出处理后的原表

## 🐛 故障排除

### 常见问题

#### 1. 编码错误
**症状**：CSV 文件读取失败，提示编码错误

**解决方案**：
- 脚本已内置多种编码自动尝试
- 如仍有问题，请将 CSV 文件另存为 UTF-8 编码

#### 2. 找不到模块
**症状**：`ModuleNotFoundError: No module named 'xxx'`

**解决方案**：
```bash
pip install 模块名
```

#### 3. 权限错误
**症状**：无法创建输出文件夹或写入文件

**解决方案**：
- 检查文件夹权限
- 以管理员身份运行（Windows）
- 使用 `chmod` 修改权限（macOS/Linux）

#### 4. 内存不足
**症状**：处理大文件时内存溢出

**解决方案**：
- 分批处理文件
- 增加系统内存
- 使用 `chunksize` 参数分块读取

## 🎯 单文件优势

### 为什么采用单文件设计？

1. **易于部署**：只需一个 Python 文件，无需管理多个依赖模块
2. **减少错误**：避免因模块路径问题导致的导入错误
3. **便于分享**：可以轻松分享给他人使用
4. **维护简单**：所有代码集中在一个文件中，便于理解和修改
5. **独立运行**：不依赖外部自定义模块，只依赖标准 Python 库和第三方库

### 内置功能模块

脚本内部集成了以下功能模块：

1. **数据诊断模块** (`修复乱码字符串`)
   - 自动修复编码问题
   - 跨系统兼容

2. **Excel工具函数模块**
   - `ifs`: 多条件判断函数

3. **主处理类** (`PaymentProcessor`)
   - 数据合并
   - 数据转换
   - 业务逻辑处理
   - 结果导出
